# Чек-лист: что внедрять после отката на a0fcfea1

Откат выполнен на коммит **a0fcfea1** (EOR: раздел в админке, столбец и меню в Отправках; stripOoo; убран столбец Перевозка в заявках).

Ниже — список изменений, которые были накатаны после него и которые нужно внедрять заново по порядку.

---

## 1. Только EOR: без Главной, только Документы и Профиль

- [ ] Если у пользователя включён только раздел EOR (нет грузов, нет других док-разделов) — не показывать вкладку «Главная» в нижней панели.
- [ ] Показывать только вкладки: Документы, Профиль.
- [ ] При открытии приложения / восстановлении вкладки не открывать «Главная» — по умолчанию открывать «Документы».
- **Файлы:** `App.tsx`, `TabBar.tsx` (onlyEorNoHome, редирект home/dashboard → docs, восстановление вкладки из localStorage).

---

## 2. Pull-to-refresh: обновление при стягивании вниз > 2 сек

- [ ] На мобильном: обновление данных только если пользователь держит стягивание вниз **больше 2 секунд** (исключить случайное срабатывание).
- [ ] Событие для обновления: например `haulz-pull-refresh`; подписка в `useDocumentsDataLoad` и при необходимости в `useCargoDataLoad`.
- **Файлы:** `App.tsx` (таймер 2 с на `.app-main`, диспатч события), хуки документов/грузов.

---

## 3. Только отправки (onlySendingsUI): интерфейс для «только служебный/EOR/отправки»

- [ ] У таких пользователей **не доступны** Главная и Грузы (вкладки скрыты).
- [ ] **Страница по умолчанию** — «Отправки» (раздел Документы).
- [ ] **Убрать** блок фильтров в разделе Отправки.
- [ ] **Убрать** суммирующее поле (инфографика/сводка по статусам и маршрутам).
- [ ] Показывать **только автомобильные отправки со статусом «В пути»**.
- [ ] **Не давать** проваливаться внутрь перевозки (клик по строке не раскрывает детали).
- **Файлы:** `App.tsx`, `TabBar.tsx`, `DocumentsPage.tsx`, `AppMainContent.tsx` (onlySendingsUI, редирект cargo → docs, скрытие Грузов и EmptyCargoState).

---

## 4. Право EOR: кто может менять статус EOR

- [ ] У всех, у кого включён раздел EOR в правах, есть право менять EOR (длинное нажатие, выбор статуса). Плюс суперадмин.
- [ ] В коде: `canEditEor = (permissions?.eor === true) || isSuperAdmin`.
- **Файлы:** `DocumentsPage.tsx`, типы (eor, isSuperAdmin).

---

## 5. Шапка: тумблер и выбор компании

- [ ] В разделе **Документы** при включённом служебном режиме в шапке **не показывать** тумблер «Служ.» и выбор компании.
- [ ] В разделе **Профиль** при включённом служебном режиме у пользователя — тоже **не показывать** тумблер и выбор компании.
- [ ] Для **onlySendingsUI** — в шапке не показывать тумблер и выбор компании (всегда).
- **Файлы:** `App.tsx` (условия отображения CustomerSwitcher и блока «Служ.»).

---

## 6. Дефолт: не заходить через Грузы / «Показать перевозки»

- [ ] Для пользователей с onlySendingsUI (или только EOR) **не открывать** по умолчанию раздел Грузы и не показывать экран с кнопкой «Показать перевозки».
- [ ] При восстановлении вкладки из localStorage: если сохранена вкладка «cargo» и у пользователя onlySendingsUI/onlyEor — подставлять «docs».
- [ ] Редирект: при activeTab = cargo и onlySendingsUI/onlyEorNoHome переключать на «docs».
- [ ] В контенте при onlySendingsUI не рендерить CargoPage и EmptyCargoState (чтобы не показывать «Показать перевозки»).
- **Файлы:** `App.tsx`, `AppMainContent.tsx` (onlySendingsUI prop, условия рендера грузов).

---

## 7. Документы: защита от падения

- [ ] В **useDocumentsDateRange**: валидация дат; при невалидных (или Invalid Date) использовать fallback `getDateRange("все")`; весь расчёт в try/catch.
- [ ] В **DocumentsPage**: инициализация дат (selectedMonthForFilter, selectedYearForFilter) с проверкой на валидность; `(items || [])` и аналогично для других массивов из хуков; не вызывать .map/.forEach по undefined.
- [ ] **dateUtils**: `parseDateOnly`, `getPayTillDate`, `getPayTillDateColor` объявлены **в начале файла** (сразу после DEFAULT_DATE_TO), чтобы при lazy-загрузке чанка Документов не было ошибки «Cannot access 'Ai' before initialization».
- **Файлы:** `src/lib/dateUtils.ts`, `src/pages/useDocumentsDateRange.ts`, `src/pages/DocumentsPage.tsx`.

---

## 8. Ошибка в разделе Документы: показ текста ошибки

- [ ] **ErrorBoundary**: fallback может быть функцией `(error, componentStack) => ReactNode`, чтобы показывать текст ошибки.
- [ ] **SectionBoundary**: передавать в fallback функцию и выводить `error?.message` под текстом «Ошибка в разделе (…)».
- **Файлы:** `ErrorBoundary.tsx`, `AppMainContent.tsx`.

---

## 9. (Опционально) Документы: не рендерить до готовности auth

- [ ] Не монтировать DocumentsPage, пока у `auth` нет `login` и `password` (показывать «Загрузка…»), чтобы не падать при частых запросах auth-registered-login.
- **Файлы:** `AppMainContent.tsx`.

---
