# Аудит адаптивности раздела «Табель учета рабочего времени»

## Контекст
Раздел находится в AdminPage, рендерится через CMSStandalonePage. Используются `app-container--allow-h-scroll`, `timesheet-container`, `timesheet-panel`, `timesheet-groups-wrap`, `timesheet-table-scroll`.

## Выявленные проблемы

### 1. Вертикальный скролл
- **Проблема**: Контент обрезается снизу (нижняя навигация/элементы частично видны).
- **Причина**: `app-container` на мобильном имеет `overflow: hidden`; `maxHeight: 70vh` у таблицы может не учитывать safe-area и высоту хедера.
- **Решение**: Добавить `padding-bottom` для safe-area; использовать `min(70vh, calc(100dvh - 280px))` для высоты таблицы.

### 2. Горизонтальный скролл
- **Проблема**: Ранее скролл не доводил до конца (дни 19–31).
- **Статус**: Исправлено через `minWidth: max-content` и `overflow-x: visible` на панелях.
- **Рекомендация**: Оставить `scrollbar-width: auto` для видимости полосы прокрутки.

### 3. Жёсткие minWidth у фильтров
- **Проблема**: `minWidth: "12rem"` (блок даты), `minWidth: "18rem"` (поиск) — на узких экранах (< 360px) вызывают горизонтальный overflow.
- **Решение**: На мобильном: `min-width: min(18rem, 100%)`, `flex: 1 1 min-content`.

### 4. Ширина колонки «Сотрудник»
- **Текущее**: `minWidth: "15rem"` — фиксированно.
- **Рекомендация**: На экранах < 400px допустить `min(12rem, 50vw)` для экономии места.

### 5. Ячейки дней (minWidth: 3.2rem)
- **Текущее**: Фиксированно, 31 день ≈ 100rem — нужен горизонтальный скролл.
- **Рекомендация**: Оставить как есть; на мобильном можно `min(2.8rem, 8vw)` для чуть меньших ячеек.

### 6. Touch targets
- **Проблема**: Кнопки «Я»/«В» ~2.2rem × 1.6rem — меньше рекомендованных 44×44px.
- **Рекомендация**: На touch-устройствах увеличить до минимум 2.75rem (~44px).

### 7. Единицы измерения
- **Хорошо**: Используются `rem`, `vh`, `%`.
- **Улучшить**: Заменить отдельные фиксированные `px` на `rem` где возможно.

### 8. Safe area
- **Проблема**: На устройствах с вырезами/округлениями контент может обрезаться.
- **Решение**: Добавить `padding-bottom: env(safe-area-inset-bottom)` для контейнера.

## Внесённые изменения (чеклист)
- [x] Адаптивная высота таблицы: `min(70vh, 600px)` → `min(60vh, 55dvh)` @768px → `min(55vh, 50dvh)` @480px
- [x] Резиновые minWidth фильтров: `min(12rem, 100%)`, `min(10rem, 100%)` для поиска
- [x] padding-bottom для safe-area в CMS: `max(2rem, env(safe-area-inset-bottom))`
- [x] Увеличенные touch targets на мобильном: `.timesheet-mark-btn` min 2.75rem @400px
- [x] Медиазапросы для breakpoints 400px, 480px, 768px
- [x] Колонка «Сотрудник» и ячейки дней: уменьшенные min-width @400px
